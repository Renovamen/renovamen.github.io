#Machine Translation

给定源语言句子 xxx，目标是最大化其对应的目标语言翻译 yyy 的概率，即：

y^=arg⁡max⁡yp(y∣x)\hat{y} = \arg \max_y p(y \mid x)y^​=argymax​p(y∣x)

#Seq2Seq

Sequence to Sequence Learning with Neural Networks. Ilya Sutskeve, Oriol Vinyals and Quoc V. Le. NIPS 2014. [Paper]

提出了 Sequence to Sequence 框架，由一个 encoder 和一个 decoder 组成。

#Encoder

一个 LSTM，用于把源语言句子 x=(x1,...,xTx)x = (x_1, ... , x_{T_x})x=(x1​,...,xTx​​) 编码成一个固定长度的向量 ccc：

ht=f1(xt,ht−1)h_t = f_1 (x_t, h_{t-1})ht​=f1​(xt​,ht−1​) c=hTxc = h_{T_x}c=hTx​​

即 LSTM 最后一个时间步输出的隐状态就是句子编码后的向量。<EOS> 是终止符，不用编码。

#Decoder

一个 LSTM，用于生成目标语言翻译 y=(y1,...,yTy)y = (y_1, ... , y_{T_y})y=(y1​,...,yTy​​)，第一个时间步的输入是 ccc，生成终止符 <EOS> 后停止句子生成。

把上述联合概率用链式法则分解后得到：

p(y)=∏t=1Tp(yt∣{y1,...,yt−1},c)p(y) = \prod_{t=1}^T p(y_t \mid \{ y_1, ..., y_{t-1} \}, c)p(y)=t=1∏T​p(yt​∣{y1​,...,yt−1​},c)

每个时间步的条件概率为：

p(yt∣{y1,...,yt−1},c)=g(yt−1,st,c)p(y_t \mid \{ y_1, ..., y_{t-1} \}, c) = g(y_{t-1}, s_t, c)p(yt​∣{y1​,...,yt−1​},c)=g(yt−1​,st​,c) si=f2(si−1,yi−1)s_i = f_2 (s_{i-1}, y_{i-1})si​=f2​(si−1​,yi−1​)

ggg 是一个非线性函数，用于输出单词 yty_tyt​ 的概率（比如 softmax），sts_tst​ 是 LSTM（decoder）在 ttt 时刻的隐状态。

#Seq2Seq + Attention

Neural Machine Translation by Jointly Learning to Align and Translate. Dzmitry Bahdanau, Kyunghyun Cho and Yoshua Bengio. ICLR 2015. [arXiv]

首次把 attention 引入 seq2seq。

#Encoder

encoder 是一个 BiLSTM，即有两个 LSTM：

  • 第一个把源句子正向输入（x1→xTxx_1 \rightarrow x_{T_x}x1​→xTx​​），所有时间步输出的隐状态为 (h1→,...,hTx→)(\overrightarrow{h_1}, ... , \overrightarrow{h_{T_x}})(h1​​,...,hTx​​​)

  • 第二个把源句子逆向输入（xTx→x1x_{T_x} \rightarrow x_1xTx​​→x1​），所有时间步输出的隐状态为 (h1←,...,hTx←)(\overleftarrow{h_1}, ... , \overleftarrow{h_{T_x}})(h1​​,...,hTx​​​)

最终，encoder 每个时间步的输出就是把 hj→\overrightarrow{h_j}hj​​ 和 hj←\overleftarrow{h_j}hj​​ 拼起来：

hj=[hj→;hj←]h_j = [\overrightarrow{h_j}; \overleftarrow{h_j}]hj​=[hj​​;hj​​]

所有时间步的输出为：

(h1,...,hTx)(h_1, ..., h_{T_x})(h1​,...,hTx​​)

#Decoder

把每个时间步的条件概率定义为：

p(yi∣y1,...,yi−1,x)=g(yi−1,si,ci)p(y_i \mid y_1, ... , y_{i-1}, x) = g(y_{i-1}, s_i, c_i)p(yi​∣y1​,...,yi−1​,x)=g(yi−1​,si​,ci​)

sis_isi​ 是 LSTM 在 iii 时刻的隐状态：

si=f(si−1,yi−1,ci)s_i = f(s_{i-1}, y_{i-1}, c_i)si​=f(si−1​,yi−1​,ci​)

cic_ici​ 是 iii 时刻的 context vector，通过把 encoder 每个时间步的输出向量加权平均得到：

ci=∑j=1Txαijhjc_i = \sum_{j=1}^{T_x} \alpha_{ij} h_jci​=j=1∑Tx​​αij​hj​

αij\alpha_{ij}αij​ 是 hjh_jhj​ 的权重，计算公式为：

eij=a(si−1,hj)e_{ij} = a(s_{i-1}, h_j)eij​=a(si−1​,hj​) αij=exp⁡(eij)∑k=1Txexp⁡(eik)\alpha_{ij} = \frac{\exp (e_{ij})}{\sum_{k=1}^{T_x} \exp (e_{ik})}αij​=∑k=1Tx​​exp(eik​)exp(eij​)​

aaa 是一个 MLP，αij\alpha_{ij}αij​ 由 eije_{ij}eij​ 归一化（softmax）后得到。相当于 αij\alpha_{ij}αij​ 代表了在生成第 iii 个目标句子单词时，第 jjj 个源句子单词的重要性。

#Text Classification

#Hierarchical Attention Network

Hierarchical Attention Networks for Document Classification. Zichao Yang, et al. NAACL 2016. [Paper]

  • 用“词-句子-文档”的层次化结构来表示一篇文档，用词向量来表示句子向量，然后用句子向量来表示文档向量。

  • 两个层次的 attention (word attention 和 sentence attention)。动机是文档中不同的句子和单词的重要性不同，且词和句子的重要性依赖于上下文。

#Word Encoder

双向 GRU：

h⇀it=GRU⇀(xit),t∈[1,T]\overrightharpoon{h}_{it} = \overrightharpoon{\text{GRU}} (x_{it}), t \in [1, T]hit​=GRU(xit​),t∈[1,T] h↼it=GRU↼(xit),t∈[T,1]\overleftharpoon{h}_{it} = \overleftharpoon{\text{GRU}} (x_{it}), t \in [T, 1]hit​=GRU(xit​),t∈[T,1] hit=[h⇀it;h↼it]h_{it} = [\overrightharpoon{h}_{it}; \overleftharpoon{h}_{it}]hit​=[hit​;hit​]

xij=Wewijx_{ij} = W_e w_{ij}xij​=We​wij​ 是文档中第 iii 个句子的第 ttt 个单词的词嵌入向量，TTT 为该句子中的单词个数。

#Word Attention

对每个 hith_{it}hit​ 计算一个权重（MLP + softmax），然后加权平均得到句子向量 sis_isi​：

uit=tanh(Wwhit+bw)u_{it} = \text{tanh} (W_w h_{it} + b_w)uit​=tanh(Ww​hit​+bw​) αit=exp⁡(uit⊤uw)∑texp⁡(uit⊤uw)\alpha_{it} = \frac{\exp (u_{it}^{\top} u_w)}{\sum_t \exp (u_{it}^{\top} u_w)}αit​=∑t​exp(uit⊤​uw​)exp(uit⊤​uw​)​ si=∑tαithits_i = \sum_t \alpha_{it} h_{it}si​=t∑​αit​hit​

softmax 中，uwu_wuw​ 是一个随机初始化的 context vector，用于表示哪些词更重要。

#Sentence Encoder

依然是双向 GRU，只不过输入为上一步得到的句子向量 sis_isi​：

h⇀i=GRU⇀(si),t∈[1,L]\overrightharpoon{h}_i = \overrightharpoon{\text{GRU}} (s_i), t \in [1, L]hi​=GRU(si​),t∈[1,L] h↼i=GRU↼(si),t∈[L,1]\overleftharpoon{h}_i = \overleftharpoon{\text{GRU}} (s_i), t \in [L, 1]hi​=GRU(si​),t∈[L,1] hi=[h⇀i;h↼i]h_i = [\overrightharpoon{h}_i; \overleftharpoon{h}_i]hi​=[hi​;hi​]

其中，LLL 为文档中的句子个数。

#Sentence Attention

对每个 hih_ihi​ 计算一个权重（MLP + softmax），然后加权平均得到文档向量 vvv：

ui=tanh(Wshi+bs)u_i = \text{tanh} (W_s h_i + b_s)ui​=tanh(Ws​hi​+bs​) αit=exp⁡(ui⊤us)∑texp⁡(ui⊤us)\alpha_{it} = \frac{\exp (u_i^{\top} u_s)}{\sum_t \exp (u_i^{\top} u_s)}αit​=∑t​exp(ui⊤​us​)exp(ui⊤​us​)​ v=∑tαihiv = \sum_t \alpha_i h_iv=t∑​αi​hi​

usu_sus​ 依然是是一个随机初始化的 context vector，用于表示哪些句子更重要。

#Document Classification

最后把文档向量 vvv 扔进 softmax 来进行分类：

p=softmax(Wcv+bc)p = \text{softmax} (W_c v + b_c)p=softmax(Wc​v+bc​)

#CNN-LSTM

模型目标：

θ∗=arg⁡max⁡θ∑(I,S)log⁡p(S∣I;θ)\theta^* = \arg \max_{\theta} \sum_{(I,S)} \log p(S \mid I; \theta)θ∗=argθmax​(I,S)∑​logp(S∣I;θ)

θ\thetaθ 是模型参数，III 是图像，SSS 是对应的正确的图像描述。也就是要最大化正确描述的概率。

为了解决解决 SSS 的长度不定的问题，用条件概率的链式法则来把上述联合概率分解成以下形式，这里为了方便扔掉了模型参数 θ\thetaθ：

log⁡p(S∣I)=∑t=0Nlog⁡p(St∣I,S0,...,St−1)\log p(S \mid I) = \sum_{t=0}^N \log p(S_t \mid I, S_0, ... , S_{t-1})logp(S∣I)=t=0∑N​logp(St​∣I,S0​,...,St−1​)

S0S_0S0​ 是起始符，SNS_NSN​ 是终止符。如果生成了终止符，则该句子生成结束。

LSTM：

在 LSTM 中，每个时间步的条件概率可以表示为：

log⁡p(St∣S1,...,St−1,I)=k(ht,zt)\log p(S_t \mid S_1, ... , S_{t-1}, I) = k(h_t, z_t)logp(St​∣S1​,...,St−1​,I)=k(ht​,zt​)

kkk 是一个非线性函数，输出为单词 StS_tSt​ 的概率。hth_tht​ 是 LSTM 在当前时间步的隐状态。ztz_tzt​ 是图像特征，在这里是 CNN 最后一个全连接层输出的一个向量；在加了 Attention 机制的 LSTM 中则是每个时间步对 CNN 最后一个卷积层输出的特征图进行 Attention 操作后得到的一个向量，依赖于 hth_tht​。

新的 xtx_txt​ 输入后，隐状态更新的公式为：

ht=f(xt,ht−1,ct−1)h_{t} = f(x_t, h_{t-1}, c_{t-1})ht​=f(xt​,ht−1​,ct−1​)

ct−1c_{t-1}ct−1​ 是 LSTM 在上一时间步的细胞状态。

图片 III 只在 t=−1t = -1t=−1 时输入一次，因为论文经过实验后发现如果每个时间步都输入一遍图像，模型会更容易过拟合和学习到图像的噪声。（这里复现了这种做法）

损失函数：

L(I,S)=−∑t=1Nlog⁡pt(St)L(I,S) = - \sum_{t=1}^N \log p_t (S_t)L(I,S)=−t=1∑N​logpt​(St​)

目标是通过调 CNN、LSTM 和 WeW_eWe​ 的参数让 Loss 最小。

训练细节：

  • 用在 ImageNet 上训练好的预训练模型来初始化 CNN 参数，效果提升明显；
  • 用在一个新闻语料库上训练好的预训练模型来初始化 WeW_eWe​ 参数，效果提升不明显；
  • 尝试 dropout 和 ensembling，效果提升了一些；
  • 调 LSTM hidden units 的数量，最后设成了 512，embedding 维度也设成了 512；
  • 除了 CNN 以外，其他部分都用 SGD 来调参，随机初始化参数，固定学习率，无 momentum;
  • 词典中只保留出现次数 > 5 的单词

首次把 Attention 机制用进 Image Captioning 中。

在上述 Encoder-Decoder 结构中，Encoder 和 Decoder 之间唯一的联系只有一个固定长度的语义向量 x−1x_{-1}x−1​，所以 Encoder 必须把原始输入的所有信息都压进 x−1x_{-1}x−1​ 中。如果原始输入包含的信息较多， x−1x_{-1}x−1​ 可能就无法表达所有信息。而且 x−1x_{-1}x−1​ 携带的信息还可能被后面输入的信息覆盖掉。

于是就有了 Attention 机制，最先把它用在 seq2seq 结构中的论文是这篇做机器翻译的：

Neural Machine Translation by Jointly Learning to Align and Translate. Dzmitry Bahdanau, KyungHyun Cho, and Yoshua Bengio. arXiv 2014. [arXiv]

#CNN

CNN 用了 VGGNet。

前面的论文是用全连接层输出的一个固定长度的向量当特征（高层特征），而这篇论文用了最后一个卷积层（用 Keras 的话说是 block5_conv3）的 H×W×DH \times W \times DH×W×D（14×14×51214 \times 14 \times 51214×14×512）的特征图来当特征（低层特征）：

a={a1,...,aL},ai∈RDa = \{ \bold{a}_1, ..., \bold{a}_L \}, \bold{a}_i \in \reals^Da={a1​,...,aL​},ai​∈RD

其中 HHH 和 WWW 为特征图的高度和宽度，DDD 为特征图的维度，L=H×WL = H \times WL=H×W。相当于对于图片的 LLL 个位置各提一个特征，每个特征都是一个 DDD 维向量（annotation vector）。

于是接下来的 LSTM 就需要在这 LLL 个位置的特征里选有用的，这就是 Attention 机制。

#Self-critical

Self-critical Sequence Training for Image Captioning. Steven J. Rennie, et al. CVPR 2017. [Paper]

Pytorch 复现：ruotianluo/self-critical.pytorch

之前的方法都是用最大似然进行语言建模，在训练时最大化模型生成的单词序列的联合概率，从而最小化交叉熵损失。这种方法存在两个问题：

  • 曝光偏差（exposure bias）：训练时用了 teacher forcing，即解码器每个时刻的输入都是训练集中的真实单词（ground truth），而测试时，解码器每个时刻的输入是自己上一时刻生成的单词，如果某一个单词预测得不够准确，之后所有单词的预测都会受到影响；

  • 训练目标和评价准则不匹配：训练时用的交叉熵损失函数，而验证时用的是 BLEU、ROUGE、METEOR、CIDEr 之类的指标，导致模型训练时无法做到充分的优化评估指标。

于是一个自然的想法是直接优化评估指标（CIDEr）。但由于生成单词的操作不可微，所以不能用一般的反向传播梯度下降来优化这些指标，因此考虑用强化学习（中的 Policy Gradient 方法）来优化。

#Policy Gradient

如果把图像描述问题看成强化学习问题：

  • agent：decoder
  • environment：单词和图像特征
  • policy：pθp_{\theta}pθ​，由模型参数 θ\thetaθ 决定
  • action：对下一个单词的预测
  • state：decoder 要更新的各种状态，如 LSTM 隐状态和细胞状态、 attention 权重等
  • reward：rrr，CIDEr 值

#SCST

论文把 baseline 定义为当前模型通过 greedy decoding 得到的句子 w^\hat{w}w^ 的 reward。所以叫 self-critical，因为 baseline 也是自己生成的，相当于自己跟自己比。于是有：
